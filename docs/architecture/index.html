2. 상태 흐름

실선 = 데이터·호출 흐름, 점선 = props/이벤트 전달. 서버 상태(Context)와 UI 상태(App 로컬)는 분리.

flowchart TD

    mount["NotesProvider 마운트"] --> fetch["api.fetchNotes()"]
    fetch --> server1[("json-server / db.json")]
    server1 --> setN["setNotes(notes)"]
    setN --> notesState["notes / loading / error
(Context state)"]
    notesState --> render["NoteList · NoteItem · NoteEditor 렌더"]
  end

  subgraph write["mutation (쓰기)"]
    direction TB
    handler["컴포넌트 핸들러
(handleSave / onDelete)"] --> action["context 액션
createNote / updateNote / deleteNote"]
    action --> apiCall["api.createNote / updateNote / deleteNote"]
    apiCall --> server2[("json-server / db.json")]
    server2 --> optimistic["setNotes(prev => ...)
함수형 낙관적 갱신"]
    optimistic --> rerender["리렌더"]
  end
