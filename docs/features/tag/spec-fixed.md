# 태그 기능 정의서 (확정본)

> [`spec-original.md`](./spec-original.md)의 기본 기능정의서를 바탕으로, 설계 단계 질의응답을
> 거쳐 데이터 구조·UI 패턴·엣지케이스를 구체화한 확정 스펙이다.

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 기능 요구사항

- 노트에 태그를 추가할 수 있다.
- 노트의 태그를 삭제할 수 있다.
- 태그 목록을 노트 상세 화면에서 확인할 수 있다.
- 태그 목록을 노트 리스트(카드)에서도 확인할 수 있다.

## 설계 결정 요약

| #   | 항목        | 확정 내용                                                           |
| --- | ----------- | ------------------------------------------------------------------- |
| 1   | 데이터 구조 | `Note`에 `tags: string[]` 인라인 (별도 엔티티 없음)                 |
| 2   | 입력 UI     | 칩(chip) 방식 · Enter/쉼표로 추가 · 칩 `x`로 삭제                   |
| 3   | 저장 시점   | 저장 버튼 클릭 시 `{title, content, tags}` 한 번에 PATCH            |
| 4   | 정규화/중복 | `trim` · 빈값 무시 · 대소문자 무시 중복 차단 · 표시는 입력 원형     |
| 5   | 표시 위치   | NoteEditor(편집) + NoteItem(읽기 전용 칩) 둘 다                     |
| 6   | 입력 제한   | 태그당 **15자** · 노트당 **10개**                                   |
| 7   | 하위 호환   | `tags` 필수 타입 + `api/notes.ts`에서 `?? []` 폴백 (db.json 미수정) |

## 상세 명세

### 1. 데이터 구조

- `Note` 인터페이스에 `tags: string[]` 필드를 추가한다 (옵셔널 아님).
- 태그는 문자열 배열로 노트 객체 안에 인라인 저장한다. 별도 `tags` 컬렉션/정규화는 하지 않는다.

```ts
interface Note {
  id: string;
  title: string;
  content: string;
  createdAt: string;
  updatedAt: string;
  tags: string[]; // ["react", "todo"]
}
```

### 2. 입력 UI (NoteEditor)

- 칩(chip) 입력 방식. 입력칸에 타이핑 후 **Enter** 또는 **쉼표(`,`)** 로 칩을 생성한다.
- 생성된 각 칩은 `x` 버튼으로 개별 삭제한다.
- 입력칸은 제목/내용 입력 아래(구분선 하단) 영역에 배치한다.

```
[react x] [todo x] [입력...        ]
Enter 또는 ,로 태그 추가
```

### 3. 저장 시점

- 태그 추가/삭제는 폼 로컬 state에만 반영하고, **저장 버튼 클릭 시** 제목/내용과 함께
  `updateNote(id, { title, content, tags })`로 한 번에 PATCH 한다.
- "취소"하면 태그 변경도 함께 롤백된다(로컬 state 기반).
- 기존 낙관적 갱신 패턴(`prev` map/spread)을 그대로 유지한다.

### 4. 정규화 / 중복 판정 (엣지케이스)

| 규칙      | 처리                                            |
| --------- | ----------------------------------------------- |
| 앞뒤 공백 | `trim()` 후 저장                                |
| 빈 태그   | trim 후 빈 문자열이면 추가하지 않음(무시)       |
| 대소문자  | `React`/`react`를 같게 보고 **중복 차단**       |
| 저장 형태 | 입력 원형 유지(예: `React` 입력 → `React` 저장) |
| 중복 입력 | 무시(추가 안 함)                                |

- 중복 비교는 소문자로 변환해 비교하되, 실제 저장/표시 값은 입력 원형을 유지한다.

### 5. 표시 위치

- **NoteEditor (상세/편집)**: 편집 가능한 칩(삭제 `x` 포함).
- **NoteItem (리스트 카드)**: 읽기 전용 칩. 날짜 표시 영역 부근에 노출한다.

```
[노트 카드]
제목
내용...
[react] [todo]   ← 읽기 전용 칩
2026-06-16
```

### 6. 입력 제한

| 항목           | 제한값        | 동작                           |
| -------------- | ------------- | ------------------------------ |
| 태그 글자 수   | 최대 **15자** | 초과 시 추가 거부              |
| 노트당 태그 수 | 최대 **10개** | 초과 시 추가 거부              |
| 허용 문자      | 제한 없음     | 공백 포함 자유 문자열(한글 등) |

### 7. 하위 호환

- 현재 `db.json`의 기존 노트에는 `tags` 필드가 없다. **db.json은 수정하지 않는다.**
- `api/notes.ts`에서 응답을 읽을 때 `tags: note.tags ?? []`로 정규화한다.
- 이로써 `Note.tags`는 필수 타입(`string[]`)으로 두되, 컴포넌트는 항상 배열을 가정할 수 있다.

```ts
// api/notes.ts 정규화 예시
return { ...note, tags: note.tags ?? [] };
```

## 비고

- 태그는 문자열로 저장한다.
- 중복 태그는 허용하지 않는다(대소문자 무시 기준).
